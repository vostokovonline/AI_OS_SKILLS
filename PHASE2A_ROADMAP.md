# 🚀 Phase 2A — Подробная Дорожная Карта

## Визуальная Панель Управления Роллаутом

---

## Фаза 0 — Pre-Flight / Baseline (0% Трафика)

### Цель
Подготовка системы к безопасному роллауту. Сбор базовых метрик для последующего сравнения.

### Длительность
1–2 дня

### Ключевые Действия
- Проверка конфигурации docker-compose.yml (ENABLE_EXECUTION_V3=false)
- Сбор базовых метрик: Success Rate, Latency (P50/P95), Stale Locks, Duplicates
- Запуск smoke tests на всех сервисах и MCP
- Проверка процедуры Rollback
- Подготовка Go/No-Go чеклиста и Cheat Sheet
- Верификация всех мониторинговых скриптов

### Навыки / MCP в Режиме Ожидания
- Legacy Execution Core — активен как бенчмарк
- Metrics Collector — сбор базовых метрик
- Rollback Handler — готов к активации

### Критерии Перехода к Фазе 1
- ✅ Все pre-flight checks пройдены
- ✅ Базовые метрики зафиксированы
- ✅ Smoke tests успешны
- ✅ Rollback процедура протестирована

## Фаза 1 — Initial Rollout (10% Трафика)

### Цель
Безопасная активация V3, сбор ранних метрик, валидация системы под нагрузкой.

### Длительность
48 часов

### Визуальный Индикатор Прогресса
```
[████████░░░░░░░░░░░░░░░░░░░░░░░░░░] 10% ████████████░░░░░░░░░░░░░░░░░░░░░░░░░
```

### Ключевые Действия
- Включение 10% трафика (EXECUTION_V3_PERCENTAGE=10)
- Активация мониторинговых скриптов:
  - check_v3_health.sh — каждые 5 минут (первые 2 часа)
  - monitor_v3_realtime.sh — каждые 30 минут
  - daily_summary_v3.sh — раз в день
- Отслеживание Go/No-Go метрик:
  - V3 Success Rate ≥ 85%
  - Delta V3 vs Legacy < 10%
  - Stale locks ≤ 3
  - Duplicates = 0
  - Traffic % в пределах 8–12%

### Навыки / MCP Активные
- Execution Core V3 — основной движок
- Goal Selector — выбор целей для исполнения
- Retry / Arbitration — логирование через Arbitration Trace
- Metrics Collector — сбор P50/P95, Success Rate
- Rollback Handler — триггер через Exit 1
- Legacy Integrator — сравнение V3 vs Legacy

### Метрики для Принятия Решения
| Метрика | Хорошо | Предупреждение | Критично | Действие |
|---------|--------|----------------|----------|----------|
| V3 Success Rate | ≥ 85% | 70–85% | < 70% | Rollback |
| Delta V3 vs Legacy | < 10% | 10–15% | > 15% | Rollback |
| Stale Locks | ≤ 3 | 3–5 | > 5 | Rollback |
| Duplicates | 0 | 0 | > 0 | Rollback |
| Traffic % | 8–12 | 5–8 / 12–15 | < 5 / > 15 | Проверить хэш |
| P50/P95 Latency | норма | небольшое отклонение | высоко | Расследовать |

### Точка Принятия Решения (48h)
- **Go** — расширение до 30%
- **No-Go** — Rollback до 0%
- **Stay** — остаться на 10% ещё 48h

## Фаза 2 — Moderate Rollout (30% Трафика)

### Цель
Расширение охвата, активация дополнительных навыков, углублённый мониторинг.

### Длительность
48–72 часа

### Визуальный Индикатор Прогресса
```
[███████████████████░░░░░░░░░░░░░░░░] 30% ████████████████████░░░░░░░░░░░░░░
```

### Ключевые Действия
- Включение Experiment Controller для контроля трафика
- Активация Skill Auto-Loader (MCP)
- Расширенный мониторинг:
  - Delta comparison по часам
  - Latency breakdown
  - Success rate per skill
- Сбор skill-specific KPIs
- Проверка автоматического подключения навыков

### Навыки / MCP Активные
- Всё из Фазы 1 +
- Experiment Controller — контроль трафика и экспериментов
- Skill Auto-Loader — динамическая загрузка навыков через MCP

### Новые Метрики для Мониторинга
- P50/P95 latency стабильна
- Skills auto-loading успешен ≥ 95%
- System load < threshold
- Queue times в пределах нормы

### Точка Принятия Решения (72h)
- **Go** — подготовка к 50%
- **No-Go** — Rollback до 10%
- **Stay** — остаться на 30% ещё 48h

## Команды Быстрого Перехода

### Запуск Фазы N
```bash
# Установить процент трафика
export EXECUTION_V3_PERCENTAGE=N

# Деплой
./deploy.sh fast

# Проверить ENV
docker exec ns_core printenv | grep EXECUTION_V3
```

### Rollback на Предыдущую Фазу
```bash
# Откат к предыдущему проценту
export EXECUTION_V3_PERCENTAGE=ПРЕДЫДУЩИЙ

# Деплой
./deploy.sh fast

# Очистить блокировки
./check_v3_health.sh --clear

# Верифицировать
./check_v3_health.sh
```

## Заключение

Данная дорожная карта представляет собой операционный план для безопасного и контролируемого роллаута Phase 2A. Каждая фаза имеет чёткие критерии перехода, метрики для мониторинга и процедуры отката.

**Ключевые принципы:**
- Безопасность превыше скорости — каждый переход требует подтверждения метрик
- Автоматизация — мониторинг и rollback автоматизированы
- Прозрачность — все метрики логируются и доступны
- Готовность к откату — на каждом этапе есть процедура возврата
