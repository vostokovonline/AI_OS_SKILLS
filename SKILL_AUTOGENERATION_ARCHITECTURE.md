# Skill Autogeneration Loop - Architecture

## Concept

Когда AI_OS встречает goal для которого нет подходящего skill, она автоматически генерирует новый навык, тестирует его и улучшает через использование.

## Ключевые компоненты

### 1. Missing Skill Detector

**Location**: `goal_executor_v2.py` → `_select_skill()`

```python
async def _select_skill(self, requirements: dict, goal_snapshot: dict):
    """
    Select skill or trigger autogeneration if missing.
    """
    # Try to find existing skill
    skill = self._find_existing_skill(requirements)

    if not skill:
        # Check if autogeneration is enabled
        if self.AUTOGENERATE_SKILLS:
            logger.warning(
                "missing_skill_detected",
                requirements=requirements,
                action="trigger_autogeneration"
            )

            # Trigger autogeneration (non-blocking)
            asyncio.create_task(
                self._autogenerate_skill(requirements)
            )

            # Return fallback skill
            return EchoSkill()

    return skill
```

### 2. Skill Autogenerator

**File**: `skill_autogenerator.py`

```python
class SkillAutogenerator:
    """
    Генерирует новые навыки на основе missing capabilities.

    Использует LLM для генерации кода.
    """

    async def generate_skill(
        self,
        missing_capabilities: List[str],
        requirements: dict
    ) -> str:
        """
        Сгенерировать код для нового навыка.

        Returns:
            skill_id сгенерированного навыка
        """

        # Step 1: Build prompt
        prompt = self._build_generation_prompt(
            missing_capabilities,
            requirements
        )

        # Step 2: Generate code via LLM
        skill_code = await self._llm_generate(prompt)

        # Step 3: Extract skill class
        skill_class = self._parse_generated_code(skill_code)

        # Step 4: Save to file
        skill_path = self._save_autogenerated_skill(
            skill_class,
            missing_capabilities
        )

        # Step 5: Validate
        is_valid = await self._validate_generated_skill(skill_path)

        if not is_valid:
            raise SkillGenerationException("Validation failed")

        # Step 6: Register
        skill_id = skill_class.id
        await self._register_skill(skill_id, skill_path)

        return skill_id

    def _build_generation_prompt(
        self,
        capabilities: List[str],
        requirements: dict
    ) -> str:
        """Build prompt for LLM code generation."""

        return f"""You are a skill generation system for AI-OS.

Generate a Python skill class with these capabilities:
{', '.join(capabilities)}

Requirements:
- Input: {requirements.get('input_type', 'text')}
- Output: {requirements.get('output_type', 'text')}
- Artifacts: {requirements.get('artifacts', 'FILE')}

The skill MUST:
1. Inherit from Skill in canonical_skills/base.py
2. Have an 'id' attribute (string, lowercase_with_underscores)
3. Have a 'version' attribute (string, default "0.1.0")
4. Implement execute(self, inputs, context) -> SkillResult
5. Return SkillResult with success, data, artifacts
6. Include proper error handling
7. Log execution steps

Example structure:

```python
from canonical_skills.base import Skill, SkillResult, Artifact

class MyGeneratedSkill(Skill):
    id = "my_generated_skill"
    version = "0.1.0"

    def execute(self, inputs, context):
        try:
            # Your logic here
            result_data = {{}}

            return SkillResult(
                success=True,
                data=result_data,
                artifacts=[
                    Artifact(
                        type="FILE",
                        content_kind="text",
                        content_location="output.txt",
                        description="Generated output"
                    )
                ]
            )
        except Exception as e:
            from logging_config import get_logger
            logger = get_logger(__name__)
            logger.error("skill_execution_failed", error=str(e))

            return SkillResult(
                success=False,
                error=str(e),
                artifacts=[]
            )
```

Generate only the Python class code, no explanation."""
```

### 3. Skill Validator

**File**: `skill_validator.py`

```python
class SkillValidator:
    """
    Валидирует автосгенерированные навыки.

    Checks:
    1. Syntax validation (import test)
    2. Structure validation (has required methods)
    3. Safety validation (no dangerous commands)
    4. Mock execution test
    """

    async def validate(self, skill_path: str) -> bool:
        """
        Run all validation checks.

        Returns:
            True if valid, False otherwise
        """

        # Check 1: Syntax
        if not self._check_syntax(skill_path):
            logger.error("skill_syntax_invalid", path=skill_path)
            return False

        # Check 2: Structure
        if not await self._check_structure(skill_path):
            logger.error("skill_structure_invalid", path=skill_path)
            return False

        # Check 3: Safety
        if not self._check_safety(skill_path):
            logger.error("skill_unsafe", path=skill_path)
            return False

        # Check 4: Mock execution
        if not await self._check_mock_execution(skill_path):
            logger.error("skill_mock_execution_failed", path=skill_path)
            return False

        return True

    def _check_safety(self, skill_path: str) -> bool:
        """
        Check for dangerous commands.

        Blocked patterns:
        - rm -rf /
        - format c:
        - DROP DATABASE
        - eval()
        - exec()
        """
        dangerous_patterns = [
            'rm -rf',
            'format c:',
            'DROP DATABASE',
            'TRUNCATE',
            '__import__("os").system',
            'eval(',
            'exec('
        ]

        with open(skill_path, 'r') as f:
            code = f.read()

        for pattern in dangerous_patterns:
            if pattern in code:
                logger.warning(
                    "dangerous_pattern_found",
                    pattern=pattern,
                    path=skill_path
                )
                return False

        return True
```

### 4. Skill Improver

**File**: `skill_improver.py`

```python
class SkillImprover:
    """
    Улучшает автосгенерированные навыки на основе execution logs.

    Trigger: After N executions or when failure rate > threshold
    """

    IMPROVEMENT_THRESHOLD_EXECUTIONS = 10
    IMPROVEMENT_THRESHOLD_FAILURE_RATE = 0.3

    async def maybe_improve_skill(self, skill_id: str):
        """
        Проверить, нужно ли улучшение, и запустить если да.
        """
        from execution_models import GoalExecutionRepository
        from infrastructure.execution_repositories import SkillStatsRepository

        async with AsyncSessionLocal() as session:
            # Get execution stats
            stats_repo = SkillStatsRepository()
            stats = await stats_repo.get(session, skill_id)

            if not stats:
                return

            # Check if improvement needed
            should_improve = (
                stats.total_executions >= self.IMPROVEMENT_THRESHOLD_EXECUTIONS and
                stats.success_rate < (1.0 - self.IMPROVEMENT_THRESHOLD_FAILURE_RATE)
            )

            if not should_improve:
                return

            logger.info(
                "skill_improvement_triggered",
                skill_id=skill_id,
                executions=stats.total_executions,
                success_rate=stats.success_rate
            )

            # Gather execution logs
            exec_repo = GoalExecutionRepository()
            executions = await exec_repo.get_by_skill(session, skill_id, limit=20)

            # Generate improved version
            improved_code = await self._generate_improvement(
                skill_id,
                executions
            )

            # Save new version
            await self._save_improved_version(skill_id, improved_code)

    async def _generate_improvement(
        self,
        skill_id: str,
        executions: List[GoalExecution]
    ) -> str:
        """
        Use LLM to analyze failures and generate improved code.
        """

        # Analyze failures
        failures = [
            e for e in executions
            if not e.success
        ][:5]  # Last 5 failures

        # Build prompt
        prompt = f"""You are a skill improver for AI-OS.

Current skill: {skill_id}

Recent failures:
{self._format_failures(failures)}

Current skill code:
```python
{await self._load_current_skill_code(skill_id)}
```

Generate an improved version that:
1. Fixes the common failure patterns
2. Adds better error handling
3. Improves logging
4. Maintains backward compatibility

Return only the improved Python class code."""

        # Generate via LLM
        improved_code = await llm_generate(prompt)

        return improved_code
```

## Database Schema

```sql
-- Autogenerated Skills Tracking
CREATE TABLE autogenerated_skills (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- Identification
    skill_id VARCHAR(255) UNIQUE NOT NULL,
    version VARCHAR(50) NOT NULL,  -- 0.1.0, 0.2.0, etc.

    -- Generation metadata
    generation_trigger VARCHAR(100),  -- missing_capability, failure_rate
    llm_model VARCHAR(100),
    generation_prompt_tokens INTEGER,

    -- Validation
    validation_status VARCHAR(50),  -- pending, passed, failed
    validation_errors TEXT,

    -- Status lifecycle
    status VARCHAR(50),  -- experimental, stable, deprecated

    -- Performance
    total_executions INTEGER DEFAULT 0,
    success_rate FLOAT,
    avg_latency_ms FLOAT,

    -- Improvement tracking
    parent_skill_id VARCHAR(255),  -- If this is improved version
    improvement_version INTEGER,  -- 1, 2, 3, ...

    -- Timestamps
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    first_used_at TIMESTAMP WITH TIME ZONE,
    last_used_at TIMESTAMP WITH TIME ZONE,
    promoted_to_stable_at TIMESTAMP WITH TIME ZONE,

    CONSTRAINT skill_status_check CHECK (
        status IN ('experimental', 'stable', 'deprecated')
    ),
    CONSTRAINT validation_status_check CHECK (
        validation_status IN ('pending', 'passed', 'failed')
    )
);

CREATE INDEX idx_autogenerated_status ON autogenerated_skills(status);
CREATE INDEX idx_autogenerated_success_rate ON autogenerated_skills(success_rate);

## Интеграция в Goal Executor

### Modified `_select_skill` method

```python
async def _select_skill(self, requirements: dict, goal_snapshot: dict):
    """
    Select skill with autogeneration fallback.
    """
    # Try existing skills
    skill = self._find_existing_skill(requirements)

    if skill:
        return skill

    # No skill found - trigger autogeneration
    logger.warning(
        "missing_skill",
        requirements=requirements,
        triggering="autogeneration"
    )

    # Trigger autogeneration (async, non-blocking)
    async def trigger_generation():
        try:
            from skill_autogenerator import skill_autogenerator

            skill_id = await skill_autogenerator.generate_skill(
                missing_capabilities=requirements.get('capabilities', []),
                requirements=requirements
            )

            logger.info(
                "skill_autogenerated",
                skill_id=skill_id
            )
        except Exception as e:
            logger.error(
                "skill_autogeneration_failed",
                error=str(e)
            )

    asyncio.create_task(trigger_generation())

    # Return fallback for this execution
    return EchoSkill()
```

## Пример работы системы

### Scenario: New Goal Type

**Goal**:
```python
{
    "title": "Analyze stock prices",
    "description": "Get stock data and generate analysis",
    "goal_type": "achievable",
    "capabilities_needed": ["stock_api", "data_analysis"]
}
```

### Flow:

1. **Goal Executor** ищет skill → не находит
2. **Trigger Autogenerator** → генерирует `stock_analyzer_v0.1`
3. **Validator** → проверяет синтаксис, safety → PASSED
4. **Registry** → регистрирует как 'experimental'
5. **Execution** → использует fallback (echo) для этого goal
6. **Next similar goal** → использует `stock_analyzer_v0.1`
7. **After 10 executions** → Skill Improver анализирует
8. **Improvement** → генерирует `stock_analyzer_v0.2`
9. **After good metrics** → промоутит в 'stable'

## Ключевые принципы

### 1. Safety First
- ✅ Синтаксическая валидация
- ✅ Safety check (dangerous commands)
- ✅ Mock execution перед реальным использованием
- ✅ Experimental status для новых навыков

### 2. Graceful Degradation
- ✅ Всегда есть fallback (echo или LLM)
- ✅ Autogeneration не блокирует execution
- ✅ Если генерация failed → система продолжает работать

### 3. Iterative Improvement
- ✅ Начинаем с stub (v0.1)
- ✅ Улучшаем через real usage
- ✅ Промоутим в stable только если metrics хорошие

### 4. Observability
- ✅ Отслеживаем все autogenerated skills
- ✅ Логируем generation prompts
- ✅ Анализируем improvement cycles

## Prevention of Skill Explosion

Эта система тоже может привести к экспоненциальному росту! Safeguards:

### 1. Generation Limits
```python
MAX_AUTOGENERATED_SKILLS = 50  # Hard cap
MIN_EXECUTIONS_BEFORE_GENERATION = 5  # Don't generate if <5 similar goals
```

### 2. Deduplication
```python
# Before generating, check for similar skills
existing = await skill_registry.find_similar(capabilities)
if existing and existing.success_rate > 0.7:
    # Use existing instead of generating new
    return existing
```

### 3. Deprecation
```python
# If autogenerated skill never used after 30 days
if skill.total_executions == 0 and skill.age > 30:
    deprecate(skill)
```
